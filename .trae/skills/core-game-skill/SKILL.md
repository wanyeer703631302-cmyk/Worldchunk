项目基础约束（不可违反）

引擎：Godot 4.x

视角：2D 俯视角

风格：像素风

类型：单机 RPG

节奏：中等、偏硬核

不允许数值堆叠解决问题

不允许 Boss 作为纯数值墙存在

一、核心设计哲学（最高优先级）
1️⃣ 世界优先于角色（World > Player）

玩家的成长主要来源于：

对世界环境的理解

对敌人行为模式的观察

对资源与属性关系的掌握

而不是：

单纯升级

装备替换

数值叠加

⚠️ 如果某个设计让“角色数值”凌驾于世界理解之上，该设计视为失败。

2️⃣ Boss 是“理解验证器”，不是数值墙

玩家可以在游戏早期就挑战 Boss

未理解机制 → 会被快速、高压地劝退

正确理解世界与规则后：

Boss 行为发生变化

压迫感下降

出现新的输出窗口

Boss 的职责是：
👉 验证玩家是否真正“读懂了这个世界”

二、系统总览（A-B-C 系统必须整体成立）
A｜世界与探索系统
World Experience System
🎯 目标

让探索世界本身，成为提升战斗能力的方式。

世界结构规则

世界由「随机模块」拼接

❌ 非纯 TileMap 平铺

✅ 模块化区域组合

每个世界模块必须包含：

地形风险（如限制移动 / 属性环境）

敌人类型倾向

资源类型倾向

世界资源设计原则

世界资源主要用于：

提高玩家容错率（生存、尝试空间）

改变战斗交互方式

提供更多“理解 → 实验 → 修正”的机会

⚠️ 世界资源 不用于直接提升 DPS

B｜战斗 & 武器系统
Action + Weapon System（核心系统）
1️⃣ 战斗基本原则

中等操作复杂度

明确攻击前摇 / 后摇

存在失败风险窗口

不追求高速连击

战斗强调判断与时机

2️⃣ 敌人攻击属性系统（必须实现）

所有敌人（小怪 / 精英 / Boss）都会释放：

带有明确属性的攻击

示例属性（可扩展）：

雷

火

冰

毒

3️⃣ 精准格挡 → 武器属性赋予（核心创新点）
触发条件

当玩家：

使用当前武器

精准格挡（Parry） 带属性的敌人攻击

结果

玩家当前武器获得该攻击对应的 属性增伤 Buff

4️⃣ Buff 严格规则（不可破坏）

Buff 类型：属性增伤

持续时间：固定（如 6–10 秒）

Buff 不可叠加

新 Buff 会 替换旧 Buff

Buff 不会永久保存

Buff 仅作用于当前武器

⚠️ Buff 是「战斗中的状态」，不是长期成长。

5️⃣ 属性克制关系（必须服务战术循环）

示例克制关系：

雷 → 水 / 机械

火 → 冰 / 植物

冰 → 雷

在多敌人战斗中，玩家应能够：

通过“吃属性 → 转化 → 克制其他敌人”形成战术循环

⚠️ 武器系统核心定位（非常重要）

武器 不解决 Boss 的核心问题

武器只影响：

输出效率

攻击节奏

操作手感

Boss 的核心解法必须来自：

世界理解

属性反制

环境准备

C｜Boss 战验证系统
Boss Validation System
Boss 战本质

❌ 不是 DPS Check

✅ 是 逻辑 Check / 理解 Check

Boss 默认状态

高压态

劝退态

攻击频率高、容错低

行为变化触发条件

当玩家：

使用正确属性

利用正确环境资源

体现对世界规则的理解

Boss 将发生变化：

攻击模式改变

压迫机制削弱

出现新的输出窗口"
